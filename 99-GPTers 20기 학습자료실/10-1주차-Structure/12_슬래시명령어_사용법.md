# 12. 슬래시 명령어 사용법 - Claude Code 기본 명령어 익히기

> **"타이핑 한 줄로 AI가 알아서 정리해줍니다"**

## 1. 🎯 개념 및 필요성 (Concept)

### 🧐 이게 뭐죠?
슬래시 명령어는 Claude Code에게 특정 작업을 자동화하도록 지시하는 **단축키**입니다. 터미널에 `/명령어`를 입력하면, AI가 알아서 적절한 폴더에 파일을 생성하고 내용을 구성합니다.

### 💡 왜 배워야 하죠?
- **자동 분류**: 파일이 올바른 폴더에 자동 저장됩니다
- **일관된 형식**: 매번 같은 템플릿으로 문서가 생성됩니다
- **시간 절약**: 폴더 찾기, 파일 만들기, 형식 맞추기 → 전부 생략

---

## 2. 🚀 명령어 목록

### 폴더-명령어 1:1 매칭

| 명령어 | 설명 | 저장 위치 |
|--------|------|-----------|
| `/21-idea-note` | 아이디어 노트 생성 | `20-created/21-ideas/` |
| `/22-reading-note` | 독서/논문 노트 | `20-created/22-reading-notes/` |
| `/33-news-briefing` | 뉴스 브리핑 | `30-collected/33-news/` |
| `/51-daily-note` | 오늘의 Daily Note | `50-periodic/51-daily/` |
| `/52-weekly-note` | 이번 주 Weekly Note | `50-periodic/52-weekly/` |

### 에이전트

| 에이전트 | 설명 | 호출 방법 |
|----------|------|-----------|
| `roadmap-builder` | 프로젝트 로드맵 대화형 설계 | "로드맵 짜줘", "프로젝트 계획 세워줘" |

---

## 3. 📋 명령어 상세 사용법

### `/51-daily-note` - 일일 기록의 시작점

매일 아침 실행하여 오늘 하루를 계획하세요.

```
/51-daily-note
```

**생성 결과**: `50-periodic/51-daily/2026-01-14.md`

---

### `/21-idea-note` - 아이디어 캡처

번뜩이는 아이디어를 놓치지 마세요.

```
/21-idea-note AI 기반 회의록 자동화 서비스
```

**생성 결과**: `20-created/21-ideas/2026-01-14 AI 기반 회의록 자동화 서비스.md`

---

### `/22-reading-note` - 독서 & 학습 기록

책, PDF, 논문에서 얻은 인사이트를 기록합니다.

```
/22-reading-note 생각에 관한 생각
```

**생성 결과**: `20-created/22-reading-notes/20260114-생각에관한생각.md`

---

### `/33-news-briefing` - 뉴스 브리핑

특정 주제의 최신 뉴스를 정리합니다.

```
/33-news-briefing AI 산업 동향
```

---

### `/52-weekly-note` - 주간 회고

한 주를 마무리하며 돌아보는 시간.

```
/52-weekly-note
```

---

### `roadmap-builder` 에이전트 - 프로젝트 관리

새 프로젝트를 시작할 때 대화형으로 로드맵을 설계합니다.

```
"가나다 프로젝트 로드맵 짜줘. 1월 12일까지 끝내야 해"
"프로젝트 계획 세워줘"
```

**주요 기능**:
- 데드라인 기반 역순 일정 계산
- Johnny Decimal 번호 체계 자동 할당 (`1X-프로젝트명/`)
- 프로젝트 정보 수집을 위한 대화형 인터페이스

**생성 결과**:
- `10-working/1X-프로젝트명/XX1-roadmap.md` - 전체 로드맵
- `10-working/1X-프로젝트명/XX2-daily-schedule.md` - 날짜별 할 일

**Daily Note 연동**: `/51-daily-note` 실행 시 오늘 할 프로젝트 태스크가 자동으로 포함됩니다!

```markdown
## 프로젝트 태스크

### 가나다 프로젝트

> 마일스톤: 문제 정의 | 데드라인까지 D-9

- [ ] 타겟 사용자 페르소나 작성
- [ ] Pain Point 3가지 정리
```

## 4. 📅 일일 워크플로우 예시

### 아침
```
/51-daily-note
```
→ 오늘 할 일 정리

### 작업 중
```
/21-idea-note [떠오른 아이디어]
/22-reading-note [책 제목]
```
→ 자동으로 적절한 폴더에 저장

### 저녁
Daily Note에서 "오늘 한 일" 정리

### 주말
```
/52-weekly-note
```
→ 이번 주 회고

## 5. 🔧 Claude Code 확장 기능 이해하기

Claude Code는 세 가지 방식으로 기능을 확장할 수 있습니다.

| 구분 | 설명 | 호출 방식 |
|------|------|----------|
| **Commands** | 저장된 프롬프트 템플릿 | `/명령어`로 명시적 호출 |
| **Skills** | Claude가 자동 감지하는 능력 | 상황에 맞게 자동 적용 |
| **Agents** | 독립 컨텍스트의 전문가 AI | 자동 또는 명시적 호출 |

### 쉽게 말하면

- **Commands** = "이 프롬프트 저장해두고 `/이름`으로 불러쓸래"
- **Skills** = "이런 상황이면 알아서 이 능력 써"
- **Agents** = "별도의 전문가를 불러서 독립적으로 일 시키기"

## 6. 🍯 꿀팁

### 명령어 번호 = 폴더 번호
명령어 앞의 숫자가 저장될 폴더를 알려줍니다.
- `/21-xxx` → `20-created/21-xxx/`
- `/22-xxx` → `20-created/22-xxx/`
- `/33-xxx` → `30-collected/33-xxx/`
- `/51-xxx` → `50-periodic/51-xxx/`

> 💡 `10-working/` 폴더는 `roadmap-builder` 에이전트가 관리합니다.

### 자연어도 가능
명령어를 모르겠으면 그냥 말해도 됩니다:
```
"오늘 데일리 노트 만들어줘"
"이 책 독서노트 만들어줘"
```
