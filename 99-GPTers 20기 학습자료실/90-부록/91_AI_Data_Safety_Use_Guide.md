## 1. 직장인이 가지는 AI 활용 걱정

### 흔한 걱정들

| 걱정 | 실제 상황 |
|------|----------|
| "내가 입력한 회사 코드가 AI 학습에 사용되나요?" | **설정에 따라 다름** - 옵트아웃 가능 |
| "경쟁사가 우리 정보를 볼 수 있나요?" | **아니요** - 다른 사용자와 데이터 공유되지 않음 |
| "회사 문서를 AI에 넣으면 영구 저장되나요?" | **아니요** - 최대 30일(기본) 또는 설정에 따라 다름 |
| "Claude Code가 내 컴퓨터 파일을 전부 읽나요?" | **아니요** - 사용자가 명시적으로 요청한 파일만 접근 |

### 이 가이드를 읽어야 하는 이유

- **정확한 정보**: Anthropic 공식 문서 기반 (2025년 8월 업데이트 반영)
- **명확한 설정 방법**: 단계별 안전 설정 가이드 제공
- **실무 적용**: 직장인 관점의 베스트 프랙티스

## 2. 핵심 요약: 안심해도 되는 이유

### ✅ Claude/Claude Code의 핵심 보안 원칙

```
┌─────────────────────────────────────────────────────────────┐
│  1. 데이터는 다른 사용자와 절대 공유되지 않습니다                 │
│  2. 기업용(Team/Enterprise/API)은 기본적으로 학습 제외          │
│  3. 개인용도 간단한 설정으로 학습 제외 가능                      │
│  4. Claude Code는 로컬에서 실행되며, 프롬프트만 전송             │
│  5. 모든 통신은 TLS 암호화로 보호                              │
└─────────────────────────────────────────────────────────────┘
```

### 사용자 유형별 한눈에 보기

| 구분 | AI 학습에 사용? | 데이터 보존 | 추천 대상 |
|------|----------------|-------------|----------|
| **Enterprise/Team** | ❌ 기본 제외 | 30일 (또는 0일) | 회사 공식 계정 |
| **API 사용** | ❌ 기본 제외 | 30일 (또는 0일) | 개발자/자동화 |
| **Pro/Max (옵트아웃)** | ❌ 제외됨 | 30일 | 개인 유료 사용자 |
| **Pro/Max (기본값)** | ⚠️ 사용됨 | 5년 | 학습 기여 희망자 |
| **Free (기본값)** | ⚠️ 사용됨 | 5년 | 일반 사용 |

## 3. 사용자 유형별 데이터 정책

### 3.1 기업용 사용자 (Team/Enterprise/API)

> **결론: 가장 안전한 옵션입니다**

#### 핵심 보호 정책

- **모델 학습 제외**: 기본적으로 AI 학습에 사용되지 않음
- **짧은 보존 기간**: 최대 30일 후 자동 삭제
- **Zero Data Retention 옵션**: API 사용 시 데이터를 아예 저장하지 않는 옵션 가능
- **관리자 제어**: 조직 차원의 데이터 정책 설정 가능

#### 예외 상황

- Development Partner Program(DPP)에 **명시적으로 opt-in**한 경우에만 학습에 사용
- 신뢰 및 안전 위반 시 별도 보존 (2~7년)

### 3.2 개인 유료 사용자 (Pro/Max)

> **결론: 설정 한 번으로 기업용과 동일한 보호 가능**

#### 기본 정책 (2025년 8월 28일 이후)

- 사용자가 **학습 허용/거부를 선택**할 수 있음
- 선택하지 않으면 기본값(학습 허용)이 적용됨

#### 옵트아웃 후 보호 수준

| 항목 | 옵트아웃 전 | 옵트아웃 후 |
|------|-----------|-----------|
| AI 학습 사용 | 사용됨 | **제외됨** |
| 데이터 보존 | 5년 | **30일** |
| 기능 제한 | 없음 | **없음** |

### 3.3 무료 사용자 (Free)

> **결론: 민감한 업무 정보는 입력하지 마세요**

- 기본적으로 AI 학습에 사용됨
- 옵트아웃 설정 가능하지만, 업무용으로는 유료 플랜 권장

## 4. 데이터 흐름 이해하기

### 4.1 Claude Code 작동 방식

```
┌──────────────────────────────────────────────────────────────────┐
│                    당신의 컴퓨터 (로컬)                          │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  Claude Code CLI                                        │    │
│  │  - 파일 읽기/쓰기: 로컬에서 처리                        │    │
│  │  - 코드 실행: 로컬에서 처리                             │    │
│  │  - 세션 캐시: 로컬에 저장 (최대 30일)                   │    │
│  └─────────────────────────────────────────────────────────┘    │
│                              │                                   │
│                              ▼ (TLS 암호화)                     │
│                     프롬프트 + 응답만 전송                       │
└──────────────────────────────────────────────────────────────────┘
                               │
                               ▼
┌──────────────────────────────────────────────────────────────────┐
│                    Anthropic 클라우드                            │
│  - AI 모델이 프롬프트 처리                                       │
│  - 응답 생성 후 반환                                             │
│  - 데이터 보존: 사용자 설정에 따름                               │
└──────────────────────────────────────────────────────────────────┘
```

### 4.2 전송되는 데이터 vs 전송되지 않는 데이터

| 전송됨 (클라우드로) | 전송 안 됨 (로컬 유지) |
|-------------------|---------------------|
| 사용자가 입력한 프롬프트 | 컴퓨터의 다른 파일들 |
| AI가 생성한 응답 | 읽기 요청하지 않은 코드 |
| 일부 메타데이터 | 시스템 정보 |
| (선택적) 에러 로그 | 로컬 실행 결과 |
* 사용자가 요구한 사항을 수행하기 위해 필요로 하는 파일 포함 전송

### 4.3 암호화 현황

| 구간 | 암호화 방식 | 설명 |
|------|-----------|------|
| 전송 중 | TLS 1.2+ | 모든 통신 암호화 |
| 저장 시 | 플랫폼별 상이 | 일부 텔레메트리는 AES-256 |

### 4.4 ⚠️ 솔직한 현실: 데이터는 서버로 전송됩니다

> **중요**: 이 섹션은 AI 사용의 현실을 정확히 이해하기 위해 반드시 읽어야 합니다.

#### 피할 수 없는 사실

| 구분 | 사실 여부 | 설명 |
|------|----------|------|
| **프롬프트가 서버로 전송됨** | ✅ 사실 | AI가 작동하려면 필수 |
| **응답도 서버에서 생성됨** | ✅ 사실 | AI 모델이 클라우드에 있음 |
| **Anthropic이 데이터를 볼 수 있음** | ✅ 사실 | 기술적으로 가능 |
| **일정 기간 저장됨** | ✅ 사실 | 최소 30일 (설정에 따라) |

#### 왜 데이터 전송이 필수인가?

```
┌─────────────────────────────────────────────────────────────────┐
│  당신의 컴퓨터                                                   │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  Claude Code CLI (로컬 실행)                              │  │
│  │                                                           │  │
│  │  "이 코드 리팩토링해줘" + 코드 내용                        │  │
│  └───────────────────────────────────────────────────────────┘  │
│                              │                                   │
│                              ▼ 반드시 전송됨 (TLS 암호화)        │
└─────────────────────────────────────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│  Anthropic 클라우드 서버                                         │
│                                                                  │
│  ✅ AI 모델이 프롬프트 처리                                      │
│  ✅ 응답 생성                                                    │
│  ⚠️ 일정 기간 저장 (30일 또는 설정에 따라)                       │
│  ❓ 학습에 사용? → 사용자 설정에 따라 다름                       │
└─────────────────────────────────────────────────────────────────┘
```

**핵심 이유**: Claude/ChatGPT 등 모든 클라우드 AI는 거대한 AI 모델(수십~수백 GB)이 클라우드 서버에 있기 때문에, 사용자의 질문을 서버로 보내야만 답변을 받을 수 있습니다.

### 4.5 "안전하다"의 진짜 의미

"안전하게 사용할 수 있다"는 것은 **"데이터가 전송되지 않는다"가 아닙니다.**

```
┌─────────────────────────────────────────────────────────────────┐
│  "안전하다"의 진짜 의미:                                         │
│                                                                  │
│  1. 다른 사용자와 공유되지 않음                                  │
│  2. 학습에 사용 여부를 선택할 수 있음 (옵트아웃)                 │
│  3. 일정 기간 후 삭제됨 (영구 저장 아님)                         │
│  4. 전송 시 암호화됨 (TLS)                                       │
│  5. Anthropic이 데이터를 판매하거나 제3자와 공유하지 않음        │
└─────────────────────────────────────────────────────────────────┘
```

### 4.6 전송 vs 저장 vs 학습 - 정확한 구분

| 단계 | 설명 | 피할 수 있나? | 방법 |
|------|------|-------------|------|
| **① 전송 (Transmission)** | 프롬프트를 서버로 보냄 | ❌ 불가능 | AI 모델이 클라우드에 있으므로 필수 |
| **② 저장 (Retention)** | 서버에 일정 기간 보관 | △ 제한적 | 최소 30일, API는 0일 가능 |
| **③ 학습 (Training)** | 미래 AI 개선에 활용 | ✅ 가능 | 옵트아웃으로 완전 제외 가능 |

#### 각 단계별 상세 설명

**① 전송 (피할 수 없음)**
- 클라우드 AI 사용의 기본 원리
- 암호화(TLS)로 전송 중 보호
- 제3자가 중간에서 볼 수 없음

**② 저장 (제한적 제어)**
- 기본 30일 저장 후 삭제
- Enterprise API: Zero Data Retention 옵션 (저장 안 함)
- 신뢰/안전 문제 발생 시 예외적으로 더 긴 보존

**③ 학습 (완전 제어 가능)**
- 옵트아웃 시 미래 AI 학습에 사용되지 않음
- 기업용(Team/Enterprise)은 기본 제외
- 옵트아웃해도 기능 제한 없음

### 4.7 완전한 프라이버시를 원한다면?

클라우드 AI가 아닌 **로컬 AI** 사용을 고려할 수 있습니다:

| 옵션 | 장점 | 단점 |
|------|------|------|
| **Ollama** | 완전 로컬, 무료, 오픈소스 | 성능 제한, 고사양 PC 필요 |
| **llama.cpp** | 가벼움, 다양한 모델 | 설정 복잡, 성능 제한 |
| **LM Studio** | GUI 제공, 사용 쉬움 | 성능 제한 |

**현실적 조언**: 로컬 LLM은 Claude 대비 성능이 낮으므로, **민감하지 않은 작업은 Claude**, **극비 정보는 로컬 LLM 또는 AI 미사용**이 현실적인 전략입니다.<br>
  또 Agent를 직접 구현해야하고, 컴퓨터 성능도 상상할 수 없는 금액을 써야 Claude Code처럼 사용가능합니다.

## 5. 안전 설정 가이드

### 5.1 [필수] 데이터 학습 옵트아웃 설정

#### 웹 (claude.ai) 설정 방법

1. **설정 페이지 접속**: [claude.ai/settings/data-privacy-controls](https://claude.ai/settings/data-privacy-controls)
2. **"Help improve Claude" 토글 찾기**
3. **토글 OFF (회색)으로 변경**
4. **즉시 적용됨** - 이후 새 세션부터 학습 제외

```
설정 경로: Settings > Privacy > "Help improve Claude" → OFF
```

### 5.2 [고수전용] Claude Code 텔레메트리 비활성화

민감한 업무 환경에서는 추가 텔레메트리도 비활성화할 수 있습니다.

#### [Windows] PowerShell 환경변수 설정

```powershell
# 일회성 설정 (현재 세션만)
$env:DISABLE_TELEMETRY = "1"
$env:DISABLE_ERROR_REPORTING = "1"
$env:DISABLE_BUG_COMMAND = "1"

# 영구 설정 (사용자 환경변수)
[Environment]::SetEnvironmentVariable("DISABLE_TELEMETRY", "1", "User")
[Environment]::SetEnvironmentVariable("DISABLE_ERROR_REPORTING", "1", "User")
[Environment]::SetEnvironmentVariable("DISABLE_BUG_COMMAND", "1", "User")
```

#### [macOS] Terminal 환경변수 설정

```bash
# 일회성 설정 (현재 세션만)
export DISABLE_TELEMETRY="1"
export DISABLE_ERROR_REPORTING="1"
export DISABLE_BUG_COMMAND="1"

# 영구 설정 (zsh 기준)
echo 'export DISABLE_TELEMETRY="1"' >> ~/.zshrc
echo 'export DISABLE_ERROR_REPORTING="1"' >> ~/.zshrc
echo 'export DISABLE_BUG_COMMAND="1"' >> ~/.zshrc
source ~/.zshrc
```

#### 모든 비필수 트래픽 한 번에 비활성화

**[Windows]**
```powershell
# 가장 강력한 프라이버시 설정
[Environment]::SetEnvironmentVariable("CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC", "1", "User")
```

**[macOS]**
```bash
# 가장 강력한 프라이버시 설정
echo 'export CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC="1"' >> ~/.zshrc
source ~/.zshrc
```

### 5.3 [고수전용]환경변수 설정 요약

| 환경변수 | 기능 | 권장 값 |
|---------|------|--------|
| `DISABLE_TELEMETRY` | 성능 메트릭 전송 비활성화 | `1` |
| `DISABLE_ERROR_REPORTING` | Sentry 에러 로그 비활성화 | `1` |
| `DISABLE_BUG_COMMAND` | /bug 명령어 비활성화 | `1` |
| `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` | 모든 비필수 트래픽 비활성화 | `1` |

### 5.4 [고수전용]로컬 캐시 관리

Claude Code는 세션 재개를 위해 로컬에 캐시를 저장합니다.

```powershell
# 캐시 위치 확인 (Windows)
# 일반적으로: %USERPROFILE%\.claude\

# 캐시 삭제 (필요 시)
Remove-Item -Recurse -Force "$env:USERPROFILE\.claude\sessions"
```

```bash
# 캐시 위치 확인 (macOS)
# 일반적으로: ~/.claude/

# 캐시 삭제 (필요 시)
rm -rf ~/.claude/sessions
```

## 6. 직장인 베스트 프랙티스

### 6.1 DO: 해도 되는 것

| 활용 사례 | 안전 수준 | 설명 |
|----------|----------|------|
| 일반 코딩 질문 | ✅ 안전 | "Python에서 리스트 정렬하는 법" |

* 사례 또한 개인 및 회사의 중요정보가 없는 것을 뜻 함.

### 6.2 매우 위험한 사용 사례 및 대안

| 위험 행동 | 위험도 | 대안 |
|----------|-------|------|
| 회사 DB 접속 정보 입력 | 🔴 위험 | 환경변수/시크릿 매니저 사용 |
| API 키/비밀번호 입력 | 🔴 위험 | 절대 입력 금지 |
| 고객 개인정보 입력 | 🔴 위험 | 가명화/익명화 후 사용 |
| 회사 기밀 문서 전체 복사 | 🟠 주의 | 필요한 부분만 추출, 민감정보 제거 |
| 경쟁사 분석 내부 자료 | 🟠 주의 | 공개 정보 기반으로 질문 |

### 6.3 민감 정보 처리 가이드

#### 입력 전 체크리스트

```
□ API 키나 비밀번호가 포함되어 있지 않은가?
□ 고객 이름, 연락처 등 개인정보가 없는가?
□ 회사 내부 시스템 URL이나 IP가 없는가?
□ 미공개 사업 전략이나 재무 정보가 없는가?
```

## 7. 자주 묻는 질문 (FAQ)

### Q1. Claude가 내 코드를 학습해서 다른 사람에게 알려주나요?

**A: 아니요, 절대 그렇지 않습니다.**

- 귀하의 데이터는 다른 사용자와 **공유되지 않습니다**
- 학습에 사용되더라도 특정 코드가 그대로 출력되지 않습니다
- 옵트아웃하면 학습 자체에서 제외됩니다

### Q2. 회사에서 Claude Code 사용해도 될까요?

**A: 회사 정책에 따라 다르지만, 적절한 설정으로 안전하게 사용 가능합니다.**

권장 순서:
1. 회사 IT/보안팀에 AI 도구 사용 정책 확인
2. Enterprise 또는 Team 플랜 도입 검토
3. 최소한 Pro + 옵트아웃 설정
4. 기밀 정보 / 민감 정보 입력 금지 원칙 준수

### Q3. 옵트아웃하면 기능이 제한되나요?

**A: 아니요, 모든 기능을 동일하게 사용할 수 있습니다.**

- AI 성능 차이 없음
- 모든 기능 정상 작동
- 단지 데이터가 미래 학습에 사용되지 않을 뿐

### Q4. 이미 입력한 데이터는 어떻게 되나요?

**A: 옵트아웃 설정은 이후 세션부터 적용됩니다.**

- 과거 데이터: 설정 변경 전 정책 적용 (삭제 요청 별도)
- 미래 데이터: 옵트아웃 정책 적용
- 개별 대화 삭제: 해당 대화는 학습에서 제외

### Q5. Claude Code가 내 컴퓨터의 모든 파일을 볼 수 있나요?

**A: 아니요, 사용자가 명시적으로 요청한 파일만 접근합니다.**

- 사용자가 읽기를 요청한 파일만 접근
- 작업 디렉토리 외 파일은 권한 요청
- 민감한 시스템 파일은 기본 제외

### Q6. API 키나 비밀번호를 실수로 입력했어요. 어떻게 하나요?

**A: 즉시 다음 조치를 취하세요:**

1. 해당 API 키/비밀번호 **즉시 변경** (가장 중요)
2. 해당 대화 삭제
3. 옵트아웃 미설정 시 즉시 설정
4. 필요시 Anthropic 지원팀에 데이터 삭제 요청

### Q7. Amazon Bedrock이나 Google Vertex AI를 통해 사용하면 더 안전한가요?

**A: 네, 기본적으로 더 강화된 프라이버시 설정이 적용됩니다.**

- 모든 비필수 트래픽 기본 비활성화
- 클라우드 제공자의 보안 정책 추가 적용
- 기업 보안 요구사항에 더 적합

### Q8. "로컬에서 실행된다"는데, 그러면 데이터가 안 나가는 거 아닌가요?

**A: 아니요, Claude Code CLI는 로컬에서 실행되지만, AI 처리를 위해 데이터는 서버로 전송됩니다.**

```
[로컬에서 처리]          [클라우드로 전송]
─────────────────        ─────────────────
• 파일 읽기/쓰기          • 프롬프트 (질문)
• 코드 실행               • AI 응답
• 터미널 명령어           • 일부 메타데이터
• 세션 캐시 저장
```

**"로컬 실행"의 의미:**
- Claude Code **프로그램 자체**가 당신 컴퓨터에서 실행됨
- 파일 접근, 코드 실행 등은 로컬에서 처리
- 하지만 **AI에게 질문할 때**는 그 내용이 서버로 전송됨

### Q9. 옵트아웃하면 데이터가 아예 전송되지 않나요?

**A: 아니요, 옵트아웃은 "학습 제외"이지 "전송 안 함"이 아닙니다.**

| 구분 | 옵트아웃 ON | 옵트아웃 OFF |
|------|-----------|-------------|
| 데이터 전송 | ✅ 전송됨 | ✅ 전송됨 |
| 서버 저장 | 30일 | 5년 |
| AI 학습 사용 | ❌ 제외 | ⚠️ 사용됨 |

**중요**: 완전히 데이터 전송을 막으려면 로컬 LLM(Ollama 등)을 사용해야 합니다.

### Q10. 그러면 결국 회사에서 Claude 사용하면 안 되는 건가요?

**A1: 회사의 정책이 있다면 반드시 회사의 정책을 따라야 합니다.**

**A2: 아니요, 적절한 주의를 기울이면 충분히 활용 가치가 있습니다.**

**현실적인 판단 기준:**

| 정보 유형 | Claude 사용 | 권장 방법 |
|----------|------------|----------|
| 일반 코딩 질문 | ✅ OK | 자유롭게 사용 |
| 공개 기술 문서 | ✅ OK | 자유롭게 사용 |
| 회사 내부 코드 (민감정보 제거) | ⚠️ 주의 | 마스킹 후 사용 |
| API 키/비밀번호 | ❌ 금지 | 절대 입력 금지 |
| 고객 개인정보 | ❌ 금지 | 익명화 또는 미사용 |
| 극비 사업 전략 | ❌ 금지 | AI 미사용 권장 |
| 회사 극비 / 개인정보 | ⚠️ 검토 | 자체 AI 개발 사용  |

**결론**: "민감 정보를 입력하지 않는다"는 원칙만 지키면 생산성 향상에 큰 도움이 됩니다.
