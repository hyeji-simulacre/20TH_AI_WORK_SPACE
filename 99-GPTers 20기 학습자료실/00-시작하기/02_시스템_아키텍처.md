## 2. 확장 기능 3종 비교

| 구분 | Commands | Skills | Agents |
|:----:|:--------:|:------:|:------:|
| **비유** | 저장된 프롬프트 | 자동 감지 능력 | 독립 전문가 |
| **호출 방식** | `/명령어` 입력 | 상황에 맞게 자동 | 자동 또는 명시적 |
| **컨텍스트** | 현재 대화 공유 | 현재 대화 공유 | 독립 컨텍스트 |
| **용도** | 문서 생성 | 데이터 수집/처리 | 복잡한 분석 |

### 쉽게 말하면

- **Commands** = "이 프롬프트 저장해두고 `/이름`으로 불러쓸래"
- **Skills** = "이런 상황이면 알아서 이 능력 써"
- **Agents** = "별도의 전문가를 불러서 독립적으로 일 시키기"

## 4. Commands (슬래시 명령어) & Agents (에이전트)

### 명령어/에이전트 → 폴더 매핑

```
┌──────────────────┐         ┌──────────────────────────────┐
│ Commands/Agents  │         │         저장 위치             │
├──────────────────┤         ├──────────────────────────────┤
│ roadmap-builder  │ ──────▶ │ 10-working/                  │
│   (에이전트)       │         │                              │
├──────────────────┤         ├──────────────────────────────┤
│ /21-idea-note    │ ──────▶ │ 20-created/21-ideas/         │
├──────────────────┤         ├──────────────────────────────┤
│ /22-reading-note │ ──────▶ │ 20-created/22-reading-notes/ │
├──────────────────┤         ├──────────────────────────────┤
│ /33-news-briefing│ ──────▶ │ 30-collected/33-news/        │
├──────────────────┤         ├──────────────────────────────┤
│ /51-daily-note   │ ──────▶ │ 50-periodic/51-daily/        │
├──────────────────┤         ├──────────────────────────────┤
│ /52-weekly-note  │ ──────▶ │ 50-periodic/52-weekly/       │
└──────────────────┘         └──────────────────────────────┘
```

### 명령어 번호 = 폴더 번호

| 명령어 앞 숫자 | 저장 폴더 |
|:-------------:|:---------|
| `/21-xxx` | `20-created/21-xxx/` |
| `/22-xxx` | `20-created/22-xxx/` |
| `/33-xxx` | `30-collected/33-xxx/` |
| `/51-xxx` | `50-periodic/51-xxx/` |

> 💡 `10-working/` 폴더는 `roadmap-builder` 에이전트가 관리합니다.
> "로드맵 짜줘" 또는 "프로젝트 계획 세워줘"라고 말하세요.

## 9. Johnny Decimal 번호 체계

```
번호대     폴더              용도
──────    ──────           ──────
 00       00-system/       시스템 설정 (템플릿, 프롬프트, 설정)
 10       10-working/      진행 중인 프로젝트
 20       20-created/      내가 만든 것
 30       30-collected/    외부에서 가져온 것
 40       40-archive/      완료/보관
 50       50-periodic/     정기 기록

 90+      99-GPTers.../    시스템/학습자료
```

### 번호 계층 구조

| 계층 | 번호 예시 | 설명 |
|------|-----------|------|
| 메인 폴더 | `10`, `20`, `30` | 십의 자리 |
| 하위 폴더 | `11`, `12`, `13`... | 10의 하위 |
| 파일 | `111`, `112`, `113`... | 11 폴더 내 파일 |

### 하위 폴더 번호 규칙

상위 폴더의 십의 자리를 따름:

```
20-created/
├── 21-ideas/           ← 20번대의 첫 번째
└── 22-reading-notes/   ← 20번대의 두 번째

30-collected/
├── 31-web-scraps/      ← 30번대의 첫 번째
├── 32-youtube/         ← 30번대의 두 번째
├── 33-news/            ← 30번대의 세 번째
├── 34-pdf-summary/     ← 30번대의 네 번째
├── 35-gdrive/          ← 30번대의 다섯 번째
└── 36-notion/          ← 30번대의 여섯 번째
```

### 파일 번호 규칙

하위 폴더 내 파일은 **3자리 번호**로 순차 할당:

```
10-working/
├── 11-project-A/
│   ├── 111_기획안.md       ← 11 × 10 + 1
│   ├── 112_요구사항.md     ← 11 × 10 + 2
│   └── 113_설계문서.md     ← 11 × 10 + 3
├── 12-project-B/
│   ├── 121_개요.md         ← 12 × 10 + 1
│   └── 122_일정.md         ← 12 × 10 + 2
└── 13-project-C/
    └── 131_초안.md         ← 13 × 10 + 1
```

> **규칙**: 폴더 번호(11) × 10 + 순번 = 파일 번호 (111, 112, 113...)

## 10. 핵심 요약

| 구성 요소 | 개수 | 역할 |
|:--------:|:----:|:-----|
| **폴더** | 6개 | 자료 분류 및 저장 |
| **Commands** | 5개 | 문서 자동 생성 |
| **Skills** | 6개 | 데이터 수집/처리 (인터랙티브 폴더 선택) |
| **Agents** | 2개 | 로드맵 설계, 노트 연결 분석 |

### 핵심 원칙

1. **출처주의**: 내가 만든 것 vs 외부에서 가져온 것
2. **번호 일관성**: 명령어 번호 = 저장 폴더 번호
3. **자동 분류**: AI가 적절한 폴더에 자동 저장
4. **설정 분리**: 유저 레벨 vs 프로젝트 레벨
