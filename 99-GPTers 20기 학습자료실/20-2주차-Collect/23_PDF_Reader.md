# 23. PDF 리더 (PDF Reader) - 잠긴 지식을 해방하라!

> **"복사도 안 되고 수정도 안 되는 PDF, 이제 내 마음대로 주무르세요."**

## 1. 🎯 개념 및 필요성 (Concept)

### 🧐 이게 뭐죠?
눈으로만 봐야 했던 PDF 문서 안의 **글자(Text)**와 **이미지(Image)**를 따로 분리해서 꺼내는 기술입니다.

### 💡 왜 배워야 하죠?
- **편집 불가 해제**: PDF는 원래 "인쇄용"이라 수정이 어렵습니다. 내용을 꺼내면 내 마음대로 요약, 편집, 재사용이 가능합니다.
- **이미지 줍줍**: 논문이나 보고서에 있는 중요한 표나 그래프 그림만 쏙쏙 뽑아서 저장할 수 있습니다.
- **AI 밥상 차리기**: Claude는 PDF를 직접 읽을 수도 있지만, 텍스트로 변환해서 주면 훨씬 더 빠르고 정확하게 분석합니다.
- **토큰 아껴쓰기**: AI가 PDF를 읽을 때는 토큰이 많이 소비됩니다. 그러니까 가능하면 우리가 직접 텍스트는 추출하고 AI는 고급분석만 시킵니다.(**500% ~ 900% 토큰 대폭 절감**)
---

## 2. 🚀 이용 방법 (How-to)

### 1️⃣ 준비물 확인
- 변환할 PDF 파일을 준비하세요. (예: `report.pdf`)
- 이 파일을 **관리하려는 폴더**에 두세요.

### 2️⃣ 명령어 입력
터미널에 아래 명령어를 입력합니다. `"파일이름.pdf"`만 바꾸세요.

#### [Windows] PowerShell
```powershell
# 예시: (현재 폴더에 있는) report.pdf 변환
python .claude\skills\pdf-reader\scripts\pdf_to_text.py "report.pdf"
```

#### [macOS] Terminal
```bash
# 예시: (현재 폴더에 있는) report.pdf 변환
python3 .claude/skills/pdf-reader/scripts/pdf_to_text.py "report.pdf"
```

### 3️⃣ 결과 확인
PDF 파일이 있던 그 자리에 2가지가 생깁니다!
1. **`report.md`**: 글자만 뽑아서 정리한 마크다운 문서
2. **`images_report/`**: PDF 안에 있던 그림들이 저장된 폴더(이미지가 있는 경우)

## 3. 🤖 프로세스 이해하기 (Process)

1. **입력 (Input)**: "이 PDF 추출해줘."하고 파일을 줍니다.
2. **분해 (Process)**: 스킬이 PDF를 한 장씩 넘기면서 스캔합니다.
   - "이건 글자네?" → 텍스트로 저장
   - "이건 그림이네?" → 이미지 파일(.png)로 저장
   - "이건 표네?" → 일반적인 표 형식 유지
3. **출력 (Output)**: 편집 가능한 마크다운(`.md`) 파일과 이미지 폴더를 선물합니다.

### 🏭 지식 관리 시스템에서의 위치
- **실행 위치**: `30-collected` 폴더에서 실행하는 것을 추천합니다. (일단 수집!)
- **이동**: 변환된 파일이 독서노트라면 `20-created/22-reading-notes/`, 프로젝트용이라면 `10-working/{프로젝트}/` 폴더로 옮겨서 관리하세요.
- **역할**: "고정된 지식(PDF)"을 "유동적인 지식(Markdown)"으로 바꿔서 내가 쓸 수 있게 만듭니다.

## 4. 💼 직무별 활용 사례 (Use Case)

### 🔬 연구원/대학원생 - "논문 파괴자"
- **상황**: 영어 논문 100편을 읽고 선행연구 조사해야 함.
- **전**: 일일이 형광펜 칠하며 읽고, 중요 그림 캡처 도구로 따느라 손목 나감.
- **후**: 논문 폴더 통째로 PDF 변환 → 텍스트는 Claude에게 "핵심 요약해줘", 그림은 자동으로 폴더에 정리됨.

### ⚖️ 법무/재무팀 - "계약서 스캐너"
- **상황**: 수백 페이지짜리 계약서에서 "독소 조항" 찾기.
- **전**: 눈 빠지게 모니터 보며 정독.
- **후**: 텍스트 추출 → 검색(Ctrl+F)으로 "배상", "해지" 키워드 1초 만에 확인.

### 🎨 콘텐츠 에디터 - "자료 수집가"
- **상황**: 정부 발행 트렌드 보고서(PDF) 내용을 인용해서 블로그 글 써야 함.
- **전**: 내용을 보고 타이핑 다시 치거나, OCR 돌리는데 오타 수정하느라 시간 낭비.
- **후**: 깔끔하게 추출된 텍스트 복사-붙여넣기, 그래프 이미지는 바로 블로그에 업로드.

### 📈 개인 투자자 D씨 - "기업 분석가"
- **상황**: 관심 있는 기업의 분기 보고서(PDF)가 수십 개임. 사업보고서를 분석하고 싶음.
- **후**: PDF 변환 → Claude에게 추출된 텍스트와 표를 줌 → **투자 지표 정리 완료**.

## 6. ⚠ 한계점

PDF 추출은 현재 레벨 3단계 수준입니다.

- 1단계: 좌->우로 텍스트 추출
- 2단계: 좌->우로 텍스트 추출 + 이미지 추출
- **3단계: 좌->우로 텍스트 추출 + 이미지 추출 + 기본 레이아웃 추출 및 특수기능(표, 다단, 페이지 이미지 처리 등)**
- 4단계: 좌->우로 텍스트 추출 + 이미지 추출 + 커스터마이징 추출(문서 1개에 전문화된 구현 특화)

4단계도 완벽 재현은 매우 어렵습니다.
하지만 현재 3단계 높은 수준으로 웬만큼의 범용 수준을 처리할 수 있습니다. 
또 첨부된 이미지 및 매우 복잡한 표는 AI에게 분석을 맡기시면 효율적으로 분석 및 해석이 가능합니다.
