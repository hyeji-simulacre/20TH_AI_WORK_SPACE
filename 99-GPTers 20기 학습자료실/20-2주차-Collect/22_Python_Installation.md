# [Windows] 사용자 가이드

이 섹션은 **Windows 10/11** 사용자를 위한 가이드입니다.  
초등학생도 따라 할 수 있도록 **하나씩, 천천히** 설명합니다.

## 1. 준비물 체크 (시작 전 확인)

- [ ] Windows 10 또는 Windows 11
- [ ] 인터넷 연결
- [ ] PC에 프로그램을 설치할 수 있는 권한 (관리자 권한)

---

## 2. 이미 설치되어 있는지 확인하기 (1분)

혹시 예전에 설치했을 수도 있으니 먼저 확인해 봅니다.

### 2-1. PowerShell(파워셸) 열기
1. 키보드에서 `Windows` 키를 누르세요.
2. `PowerShell`이라고 입력하세요.
3. 검색 결과에서 **Windows PowerShell**을 클릭하세요.

### 2-2. 버전 확인 명령어 입력
PowerShell 창에 아래 명령어를 입력하고 `Enter`를 누르세요.

```powershell
python --version
```

- **성공(설치됨)**: `Python 3.12.x` 처럼 버전 숫자가 나옵니다. -> **[6. 다음 단계](#6-다음-단계설치가-끝났다면-여기까지-하면-더-좋아요)**로 이동하세요.
- **실패(설치안됨)**: 빨간 글씨 에러나 아무 반응이 없다면 아래 **[3. 설치하기]**로 넘어가세요.

---

## 3. Python 다운로드 및 설치 (중요!)

### 3-1. 다운로드
1. 브라우저 주소창에 `https://www.python.org/downloads/release/python-31210/` 입력
2. 노란색 큰 버튼 **`Download Python 3.12.x`** 클릭
   - (버전 숫자가 조금 달라도 3.12로 시작하면 괜찮습니다)

### 3-2. 설치 파일 실행 (제일 중요 ⭐)
다운로드 받은 파일(`python-3.12.x-amd64.exe`)을 실행합니다.

> **⚠️ 잠깐! 여기서 멈추세요!**  
> 설치 첫 화면 맨 아래에 있는 체크박스를 꼭 확인하세요.

- [x] **Add python.exe to PATH**  
  (이걸 체크 안 하면 나중에 명령어를 못 찾아서 고생합니다. **꼭 체크하세요!**)

체크했다면 **`Install Now`**를 눌러 설치를 완료하세요.

---

## 4. 설치 결과 확인

PowerShell을 **새로 열어서** (기존 창은 닫고 다시 여세요) 확인합니다.

```powershell
python --version
```
`Python 3.12.x`라고 나오면 성공입니다! 🎉

---

## 5. 자주 묻는 질문 (Troubleshooting)

### Q. `python` 쳤더니 마이크로소프트 스토어가 열려요.
**해결법**: 윈도우 설정 > 앱 > 앱 실행 별칭 > `python.exe`와 `python3.exe`를 찾아서 **끔(Off)**으로 바꾸세요.

### Q. `찾을 수 없습니다` 에러가 떠요.
**해결법**: 설치할 때 `Add to PATH` 체크를 깜빡하신 것 같습니다. 제어판에서 Python을 삭제하고 처음부터 다시 설치하면서 **체크박스를 꼭 체크**해주세요.

# [macOS] 사용자 가이드

이 섹션은 **Mac(MacBook, iMac, Mac mini)** 사용자를 위한 가이드입니다.  
Mac은 시스템 구조가 조금 다르므로 터미널 사용법을 잘 익혀두시면 좋습니다.

## 1. 준비물 체크

- [ ] macOS (Intel 또는 Apple Silicon M1/M2/M3 모두 가능)
- [ ] 인터넷 연결
- [ ] 관리자 암호 (Mac 켤 때 입력하는 비밀번호)

---

## 2. 이미 설치되어 있는지 확인하기

Mac에는 기본적으로 Python이 있을 수 있지만, 버전이 중요합니다.

### 2-1. 터미널(Terminal) 열기
1. `Command(⌘) + Space`를 눌러 **Spotlight**를 엽니다.
2. `Terminal`을 입력하고 `Enter`를 칩니다.

### 2-2. 버전 확인 명령어 입력
터미널에 아래 명령어를 입력하세요. (**주의**: Mac은 `python` 대신 `python3`라고 입력해야 할 때가 많습니다)

```bash
python3 --version
```

- **성공**: `Python 3.12.x`가 나온다면 -> **[4. 설치 확인]**으로 넘어가세요.
- **구버전**: `Python 3.9.x` 이하라면 최신 버전 설치를 권장합니다.
- **실패**: `command not found`가 뜬다면 설치가 필요합니다.

---

## 3. Python 설치하기 (공식 설치 패키지)

초보자에게는 `Homebrew`보다 공식 홈페이지의 설치 패키지(`.pkg`) 방식이 가장 확실하고 오류가 적습니다.

### 3-1. 다운로드
1. [Python macOS 다운로드 페이지](https://www.python.org/downloads/release/python-31210/) 접속
2. **Stable Releases** 아래에서 최신 `Python 3.12.x` 버전을 찾습니다.
3. **`macOS 64-bit universal2 installer`**를 클릭하여 다운로드합니다.

### 3-2. 설치 진행
1. 다운로드된 `.pkg` 파일을 더블 클릭합니다.
2. `계속(Continue)` > `동의(Agree)` > `설치(Install)`를 누릅니다.
3. Mac **로그인 비밀번호**나 **Touch ID**를 입력하여 승인합니다.
4. **중요**: 설치 마지막에 폴더가 하나 열리면서 `Install Certificates.command` 라는 파일이 보일 수 있습니다.
   - 이 파일을 더블 클릭해서 실행해 주세요. (인터넷 통신 보안 인증서를 설치하는 과정입니다. 까만 창이 떴다가 'Process completed' 하면 닫으면 됩니다.)

---

## 4. 설치 확인 및 환경 설정

터미널을 **완전히 종료(Command + Q)** 했다가 다시 열어주세요.

```bash
python3 --version
pip3 --version
```

둘 다 버전 정보가 잘 나온다면 설치 성공입니다!

---

## 5. 자주 묻는 질문 (Troubleshooting)

### Q. `python`이라고 치면 안 되고 `python3`라고 쳐야 되나요?
**답변**: 네, Mac에서는 기본적으로 `python`은 옛날 버전(2.x)을 가리키거나 비어있고, 우리가 설치한 최신 버전은 `python3` 명령어를 씁니다. 앞으로 모든 명령어에서 `python` 대신 `python3`, `pip` 대신 `pip3`를 쓴다고 생각하면 편합니다.

### Q. `Permission denied`(권한 거부) 에러가 떠요.
**해결법**: 시스템 폴더를 건드리려 할 때 발생합니다. 명령어 앞에 `sudo`를 붙여보세요 (예: `sudo pip3 install ...`). 하지만 가상환경(`.venv`)을 쓰면 이런 문제가 거의 없습니다.

# [공통] 다음 단계: 가상환경 만들기

설치가 끝났다면, 이제 본격적으로 작업 환경을 만들어 봅시다.  
이 과정은 Windows와 Mac이 거의 비슷하지만, 명령어가 조금 다릅니다.

> **💡 참고: 가상환경, 꼭 써야 하나요?**  
> - **가상환경 권장**: 개발자 관점에서는 프로젝트마다 버전과 라이브러리를 독립적으로 관리하기 위해 가상환경 사용을 **강력 권장**합니다.
> - **전역 설치(선택)**: 만약 "나는 이 컴퓨터 혼자 쓰고, 매번 켜기 귀찮다" 하시는 분은 가상환경 생성 단계를 건너뛰고 바로 `pip install`로 스킬을 설치해도 됩니다. (단, 나중에 버전 꼬임 문제가 발생할 수 있습니다.)

### 1단계: 워크스페이스 폴더로 이동

터미널(PowerShell/Terminal)을 열고, 본인의 워크스페이스 경로로 이동합니다.

**[Windows]**
```powershell
# 예시 경로 (본인의 실제 폴더 경로로 확인 후 입력하세요)
cd "D:\gpters\2.20기AI워크스페이스\20THAIWORKSPACE-HJ\YOUR-WORKSPACE-v0.8"
```

**[macOS]**
```bash
# 예시 경로 (본인의 실제 폴더 경로로 확인 후 입력하세요)
cd ~/Desktop/YOUR-WORKSPACE-v0.8
```

### 2단계: 가상환경 생성 (최초 1회)

**[Windows]**
```powershell
python -m venv .venv
```

**[macOS]**
```bash
python3 -m venv .venv
```

### 3단계: 가상환경 활성화 (매번 작업 시작할 때)

터미널을 새로 열 때마다 이 명령어로 "작업 모드"를 켜야 합니다. (전역 설치 사용자는 생략 가능)

**[Windows]**
```powershell
.\.venv\Scripts\Activate.ps1
```
*(만약 빨간색 보안 오류 글씨가 뜨면 `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` 를 한 번 입력해 주세요)*

**[macOS]**
```bash
source .venv/bin/activate
```

> **확인 방법**: 터미널 입력창 맨 앞에 초록색(또는 괄호)으로 `(.venv)`가 생겼나요?  
> 그렇다면 준비 완료입니다! 이제 스킬 사용법을 배우러 가볼까요? 🚀
