# 32. 구글 드라이브 연동 (Google Drive Integration) - 무한한 자산 저장소

> **"노션이 '서재'라면, 구글 드라이브는 '창고'입니다. 무엇이든 던져두세요."**

## 1. 🎯 개념 및 필요성 (Concept)

### 🧐 이게 뭐죠?
내 컴퓨터의 파일(문서, 이미지, 영상 등)을 구글 드라이브로 보내거나 가져오는 기술입니다. 드라이브 설치형 프로그램(Google Drive for Desktop) 없이도, **명령어 하나로** 파일을 주고받습니다.

### 💡 왜 배워야 하죠?
- **대용량 파일 관리**: 노션은 파일 크기와 형식에 제한이 있지만, 구글 드라이브는 (거의) 제한이 없습니다. 문서,영상, 고화질 이미지 원본은 여기에 보관합니다.
- **자동 분류**: AI에게 "업로드해줘"라고 하면, 알아서 업로드하고 성공/실패 여부도 표시합니다.(`(fail)_` or `(done)_`).
- **자산(Asset) 아카이빙**: 프로젝트가 끝나면 결과물들을 통째로 드라이브에 박제해서 보관할 수 있습니다.

---

## 2. 🚦 단계별 접근 (Levels)

노션과 마찬가지로 안전한 연동을 위해 단계를 나눕니다.

| 레벨 | 기능 | 방향 | 특징 | 
|---|---|---|---|
| **Lv1 (초급)** | **단방향 업로드** | 💻 로컬 → ☁️ 구글 | 백업, 아카이빙, 원본 저장 목적. |
| **Lv2 (중급)** | **분리된 양방향** | 💻 로컬 ↔️ ☁️ 구글 | 필요한 자료 검색 후 로컬로 가져오기. |

> **"로컬은 작업장, 구글 드라이브는 창고"라는 개념을 잊지 마세요!**

---

## 3. 🏭 지식 흐름 설계 (Workflow)

구글 드라이브는 지식 관리 시스템에서 **"장기 보존 서고(Archive)"**입니다.

- **Lv1 (Up)**: 로컬 폴더 → `Google Drive` (AI가 "어느 폴더의 파일을 업로드할까요?" 질문)
- **Lv2 (Down)**: `Google Drive` → 프로젝트 폴더 (AI가 "어느 프로젝트에 저장할까요?" 질문)
- **Tip1**: 로컬 용량이 부족하면 과감하게 드라이브로 보내고 로컬은 비우세요 (`Empty`).
- **Tip2**: 팀과 파일공유를 할 때 활용하세요.

---

## 3. 🚀 Lv1: 업로드 (Upload) - "창고에 넣기"

### 1️⃣ 준비물
- 백업할 파일을 준비합니다.
- AI에게 "어느 폴더의 파일을 업로드할까요?"라고 물으면, 폴더를 지정해주면 됩니다.

### 2️⃣ 명령어 입력
#### [Windows] PowerShell

AI에게 지시하세요

```powershell
# 지정 폴더의 파일을 구글 드라이브로 전송
python .claude\skills\gdrive-down-lv2\scripts\gdrive_upload.py
```

#### [macOS] Terminal
```bash
# 지정 폴더의 파일을 구글 드라이브로 전송
python3 .claude/skills/gdrive-down-lv2/scripts/gdrive_upload.py
```

### 3️⃣ 결과 확인
- 파일이 사라지지 않습니다. (놀라지 마세요!)
- 업로드한 폴더에서 파일명만 변경됩니다.
- 성공 시: `(done)_파일명`
- 실패 시: `(fail)_파일명`
- 구글 드라이브에 가보면 파일이 잘 들어와 있습니다.

### 4️⃣ (고급) 특정 폴더/파일만 콕 집어 올리기
내가 원하는 파일만 딱 골라서 원하는 폴더로 보낼 수도 있습니다.

```powershell
# 1. 특정 드라이브 폴더로 업로드
python .claude/skills/gdrive-down-lv2/scripts/gdrive_upload.py --select-uploadpage-id "구글드라이브_폴더ID_또는_URL"

# 2. 특정 파일만 선택해서 업로드 (파일명 지정)
python .claude/skills/gdrive-down-lv2/scripts/gdrive_upload.py --files "중요보고서.pdf" "사진.jpg"
```

---

## 4. 🚀 Lv2: 다운로드 (Download) - "창고에서 꺼내기"

### 1️⃣ 명령어 입력
#### [Windows] PowerShell
```powershell
# 구글 드라이브 내용을 프로젝트 폴더로 다운로드
python .claude\skills\gdrive-down-lv2\scripts\gdrive_download.py
```

#### [macOS] Terminal
```bash
# 구글 드라이브 내용을 프로젝트 폴더로 다운로드
python3 .claude/skills/gdrive-down-lv2/scripts/gdrive_download.py
```

### 2️⃣ 결과 확인
- 실행 시 폴더 선택 메뉴가 표시됩니다. 프로젝트별 저장(`10-working/{프로젝트명}/`)이 권장됩니다.
- 선택한 폴더에 파일이 다운로드됩니다.

### 3️⃣ (고급) 원하는 것만 쏙 골라오기
모든 파일을 다 가져오는 게 부담스럽다면, 필요한 것만 지정해서 가져오세요.

```powershell
# 1. 파일 이름으로 하나만 다운로드 (중복 시 선택 창 뜸)
python .claude/skills/gdrive-down-lv2/scripts/gdrive_download.py "2024_매출보고서.pdf" "C:\Users\User\Desktop"

# 2. 특정 폴더만 통째로 다운로드
python .claude/skills/gdrive-down-lv2/scripts/gdrive_download.py "프로젝트A_폴더명" "C:\Users\User\Desktop\ProjectA"

# 3. 폴더 ID나 URL로 확실하게 다운로드
python .claude/skills/gdrive-down-lv2/scripts/gdrive_download.py --url "https://drive.google.com/..." "C:\Downloads"
```

## 5. 💼 직무별 활용 사례 (Use Case)

### 🎨 디자이너 - "디자인 소스 보물창고"
- **상황**: PSD, AI 원본 파일 용량이 너무 커서 메일로 못 보냄.
- **전**: 드라이브 웹 들어가서 드래그&드롭 기다리기... (가끔 멈춤)
- **후**: 작업 끝난 파일을 AI에게 "드라이브에 업로드해줘"라고 하면 퇴근. 밤새 알아서 업로드 완료.

### 📸 포토그래퍼/영상 편집자 - "원본 백업 머신"
- **상황**: SD카드에 있는 수백 기가 촬영본 백업.
- **전**: 외장 하드 연결하고 복사 붙여넣기 노가다.
- **후**: 스크립트로 "SD카드 내용 전부 드라이브 특정 폴더로 백업해" 명령 한 번으로 끝.

### 📊 데이터 분석가 - "데이터 공유 허브"
- **상황**: 팀원들이 수집한 엑셀(CSV) 파일들을 한곳에 모아 분석해야 함.
- **전**: 각자 메일로 보내고 취합하느라 버전 꼬임.
- **후**: 각자 로컬에서 `Upload` 실행 → 구글 드라이브 한 폴더에 모임 → `Download`로 내 로컬에 받아 분석 시작.

### 🎬 유튜버/영상 편집자 D씨 - "편집 소스 무한 리필"
- **상황**: 효과음, 자막 템플릿, 무료 이미지 등을 모아두는데 컴퓨터 용량이 부족함.
- **후**: 수집한 소스는 AI에게 "드라이브에 업로드해줘"라고 해서 저장. 편집할 때만 필요한 폴더를 `Download`해서 쓰고 다시 삭제. (내 컴퓨터 용량 확보!)

## 6. 🍯 꿀팁: 팀원과 함께 쓰려면?

구글 드라이브의 **"공유 드라이브(Shared Drive)"**를 활용하세요.
1. 환경 설정(`.env`)에서 `GDRIVE_UPLOAD_DEFAULT_FOLDER_ID`를 **팀 공유 폴더의 ID**로 설정합니다.
2. 팀원 A가 올리면(`Upload`), 팀원 B가 바로 받을 수(`Download`) 있습니다.
3. **"우리 팀만의 파일 전송 터널"**이 뚫리는 셈이죠!


---

**💎 Tip**: 구글 드라이브 연동은 **"일단 던져두면 정리된다"**는 마음으로 쓰면 편합니다. 파일이 너무 많아 정리하기 귀찮을때, 일단 싹 다 드라이브로 올려버리세요!
