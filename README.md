## 스터디 소개

이 워크스페이스는 **Claude Code**를 활용한 지식관리 시스템입니다.

> "자료 찾느라 업무할 시간을 다 놓치고 계신가요?"
>
> 쌓인 회의록, 주간보고, 할 일 정리 문서에 치여 사는 분들을 위해
> **슬래시 명령어 하나로 자동 분류되는 AI 워크스페이스**를 만들었습니다.

### 이 워크스페이스로 할 수 있는 것

- **자동 정리**: 내가 지시만 하면 자료가 알맞은 폴더에 자동 저장
- **즉시 검색**: 나만의 AI가 내 자료를 즉시 검색하고 연관 자료까지 추천
- **자동 작성**: 문서 검색과 정리는 물론, 원하는 내용까지 작성

**활용 예시:**
- 기획 회의 전: "OO관련 자료 모아 정리해줘" → 2분 내 브리핑 자료 완성
- 주간보고 작성: "이번 주 내가 한 일 정리해줘" → 야근 탈출
- 콘텐츠 기획: "최근 트렌드 적용해줘" → 트렌드 + 내 업무 기반 신규 기획안

### 공동 스터디장

| 이름 | 소개 |
|------|------|
| **안상영** | 디지털 사업 전문가 · 노에스이오 테크놀로지 대표 · 자체 AI Agent 개발 및 데이터 수집/분석 설계 |
| **정혜지** | 기록관리 전문가(아키비스트) · ContextA 대표 · 브런치 칼럼니스트 · 서울대/한국외대/명지대 기록대학원 강의 |

> **기록관리 전문가의 '지식 구조'** × **디지털 전문가의 '자동화 기술'** 결합

## 시작하기

### 1단계: 사전 준비 확인

- [ ] Claude Pro 구독 완료 ($20/월)
- [ ] VSCode 설치 완료
- [ ] Claude Code 설치 완료

> 설치 안내: [Claude Code 공식 가이드](https://docs.anthropic.com/ko/docs/claude-code/getting-started)

### 2단계: 템플릿 설정

1. **압축 해제**
   - 다운로드한 ZIP 파일을 원하는 위치에 압축 해제
   - 예: `~/Documents/AI-Workspace/`

2. **VSCode로 폴더 열기**
   - VSCode 실행 → File → Open Folder
   - `AI-Workspace-Template` 폴더 선택

3. **Claude Code 실행**
   - 터미널 열기: `Ctrl+`` (백틱) 또는 Terminal → New Terminal
   - 입력: `claude`

### 3단계: 첫 명령 실행

```
/51-daily-note
```

오늘 날짜의 Daily Note가 생성됩니다!

## 폴더 구조 설명

| 폴더 | 한 줄 설명 |
|------|-----------|
| `00-system/01-templates/` | 내가 지정하고 AI가 참조하는 템플릿 |
| `10-working/` | 지금 하는 것 (진행 중인 프로젝트) |
| `20-created/` | 내가 만든 것 (내 생각, 메모, 초안) |
| `30-collected/` | 남에게서 가져온 것 (스크랩, 레퍼런스, PDF) |
| `40-archive/` | 보관 (끝난 것) |
| `50-periodic/` | 주기적 기록 (일기, 회고) |

### 왜 00번이 Inbox가 아닌가?

일반적인 PKM에서는 `00-inbox`에 미분류 문서를 넣고, 나중에 분류합니다.
하지만 **AI 에이전트가 자동 분류**하는 이 워크스페이스에서는 "미분류" 상태가 불필요합니다.
대신 AI가 참조할 템플릿을 넣어두면 더 유용합니다.

### 왜 5개인가?

PKM 시맨틱 검색 시, 우선순위별 벡터 스토어를 5~7개로 나누면 효율적입니다.
이 구조는 그대로 검색 우선순위로 활용할 수 있어, 확장 시에도 고민 없이 적용됩니다.

> 필요에 따라 `temp/` 등 커스텀 폴더를 추가해도 됩니다.

## Claude Code 확장 기능 이해하기

Claude Code는 세 가지 방식으로 기능을 확장할 수 있습니다.

| 구분 | 설명 | 호출 방식 |
|------|------|----------|
| **Commands** | 저장된 프롬프트 템플릿 | `/명령어`로 명시적 호출 |
| **Skills** | Claude가 자동 감지하는 능력 | 상황에 맞게 자동 적용 |
| **Agents** | 독립 컨텍스트의 전문가 AI | 자동 또는 명시적 호출 |

### 쉽게 말하면

- **Commands** = "이 프롬프트 저장해두고 `/이름`으로 불러쓸래"
- **Skills** = "이런 상황이면 알아서 이 능력 써"
- **Agents** = "별도의 전문가를 불러서 독립적으로 일 시키기"

> 이 워크스페이스의 `/51-daily-note`, `/21-idea-note` 등은 Commands 형태로 호출하지만, Skill처럼 자동화된 워크플로우를 수행합니다.

**공식 문서:**
- [Slash Commands](https://docs.anthropic.com/en/docs/claude-code/slash-commands)
- [Skills](https://docs.anthropic.com/en/docs/claude-code/skills)
- [Sub-agents](https://docs.anthropic.com/en/docs/claude-code/sub-agents)

## Claude Code 설정 레벨 (유저 vs 프로젝트)

Claude Code 설정은 **유저 레벨**(홈 디렉토리)과 **프로젝트 레벨**(작업 폴더)로 나뉘며, 프로젝트 레벨이 우선 적용됩니다.

| 구분 | 유저 레벨 | 프로젝트 레벨 |
|------|-----------|---------------|
| **경로** | `~/.claude/` | `프로젝트폴더/.claude/` |
| **적용 범위** | 모든 프로젝트 | 해당 프로젝트만 |
| **비유** | 나의 정체성 | 이 작업 공간의 규칙 |
| **팀 공유** | 불가 (개인용) | 가능 (Git 커밋) |

### 어디에 뭘 넣을까?

**유저 레벨에 넣을 것** (나를 따라다니는 것)
- 내 응답 스타일 선호 ("반말로", "이모지 쓰지마")
- 범용 스킬 (Google Calendar, 웹 검색 등)
- API 키 같은 인증 정보

**프로젝트 레벨에 넣을 것** (이 프로젝트 전용)
- 폴더 구조 설명
- 프로젝트 전용 스킬 (`/51-daily-note`, `/21-idea-note` 등)
- 글쓰기 스타일 가이드

> 이 워크스페이스의 `.claude/` 폴더에는 프로젝트 전용 설정이 들어있습니다.
> 개인 선호도는 `~/.claude/`에 따로 설정하세요.

**공식 문서:**
- [Settings](https://docs.anthropic.com/en/docs/claude-code/settings)
- [Memory (CLAUDE.md)](https://docs.anthropic.com/en/docs/claude-code/memory)

## 슬래시 명령어 (Commands)

### 슬래시 명령어란?

슬래시 명령어는 **자주 반복하는 작업을 한 줄로 실행하는 단축키**입니다.
마치 스마트폰의 단축어 앱처럼, 복잡한 작업을 `/명령어` 한 번으로 자동화합니다.

**슬래시 명령어가 해주는 일:**
1. 올바른 폴더 위치를 찾아 파일 생성
2. 미리 정의된 템플릿 구조 적용
3. 날짜, 번호 등 메타데이터 자동 입력
4. 필요한 경우 외부 정보 수집 (뉴스, 검색 등)

**왜 사용하나요?**
- 매번 "어디에 저장하지?" 고민할 필요 없음
- 일관된 포맷으로 문서 관리 가능
- AI가 맥락을 이해하고 적절한 내용 구성

### 명령어 목록

| 명령어 | 설명 | 저장 위치 | 필요 입력 |
|--------|------|-----------|-----------|
| `/21-idea-note [제목]` | 아이디어/기획 노트 생성 | `20-created/21-ideas/` | 아이디어 제목 |
| `/22-reading-note [책 제목]` | 독서/PDF/논문 노트 생성 | `20-created/22-reading-notes/` | 책/자료 제목 |
| `/33-news-briefing [주제]` | 뉴스 브리핑 정리 | `30-collected/33-news/` | 검색 키워드 |
| `/51-daily-note` | 오늘의 Daily Note 생성 | `50-periodic/51-daily/` | 없음 (자동) |
| `/52-weekly-note` | 이번 주 Weekly Note 생성 | `50-periodic/52-weekly/` | 없음 (자동) |

### 명령어 상세 설명

#### `/21-idea-note` - 아이디어 캡처

**언제 사용하나요?**
- 회의 중 번뜩이는 아이디어가 떠올랐을 때
- 샤워하다가, 산책하다가 영감이 왔을 때
- "이거 나중에 해봐야지" 싶은 순간

**사용 방법:**
```
/21-idea-note AI 기반 회의록 자동화 서비스
```

**생성되는 파일:**
- 위치: `20-created/21-ideas/2026-01-08-AI-기반-회의록-자동화-서비스.md`
- 파일명 형식: `YYYY-MM-DD-제목.md`

**템플릿 구조 예시:**
```markdown
# AI 기반 회의록 자동화 서비스

## 핵심 아이디어
(AI가 입력한 제목을 바탕으로 아이디어 정리)

## 문제 정의
(이 아이디어가 해결하려는 문제)

## 예상 결과물
(구현했을 때 기대 효과)

## 다음 단계
(실행을 위한 첫 번째 액션)
```

**팁:**
- 제목은 짧고 명확하게 (검색하기 쉽도록)
- 완벽하지 않아도 OK, 일단 기록하는 게 중요
- 나중에 `zettelkasten-linker`로 관련 노트 연결 가능

#### `/22-reading-note` - 독서 & 학습 기록

**언제 사용하나요?**
- 책을 읽고 핵심 내용을 정리할 때
- PDF 문서나 논문을 분석할 때
- 온라인 강의를 듣고 요약할 때

**사용 방법:**
```
/22-reading-note 생각에 관한 생각
```

**생성되는 파일:**
- 위치: `20-created/22-reading-notes/2026-01-08-생각에-관한-생각.md`

**템플릿 구조 예시:**
```markdown
# 생각에 관한 생각

## 기본 정보
- 저자:
- 출판년도:
- 읽은 날짜: 2026-01-08

## 한 줄 요약
(이 책/문서를 한 문장으로)

## 핵심 인사이트
(가장 기억에 남는 3가지)

## 적용 포인트
(내 일/삶에 어떻게 적용할 수 있는가)

## 인용구
(기억하고 싶은 문장들)
```

**팁:**
- PDF 파일이 있다면 `pdf-reader` 스킬이 자동으로 내용 추출
- "이 책이랑 비슷한 내 노트 찾아줘"로 연결 가능

#### `/33-news-briefing` - 뉴스 브리핑

**언제 사용하나요?**
- 업계 동향을 빠르게 파악하고 싶을 때
- 회의 전 관련 뉴스를 정리해야 할 때
- 특정 주제의 최신 정보가 필요할 때

**사용 방법:**
```
/33-news-briefing AI 산업 동향
```

**생성되는 파일:**
- 위치: `30-collected/33-news/2026-01-08-AI-산업-동향.md`

**실행 과정:**
1. 웹 검색으로 최신 뉴스 수집
2. 주요 기사 3-5개 선별
3. 핵심 내용 요약 및 시사점 정리

**템플릿 구조 예시:**
```markdown
# AI 산업 동향 브리핑

> 생성일: 2026-01-08

## 주요 뉴스

### 1. [기사 제목]
- 출처: OO일보
- 요약: ...
- 시사점: ...

## 종합 분석
(전체적인 트렌드와 의미)

## 내 업무 연관성
(이 소식이 내 일과 어떻게 연결되는지)
```

**팁:**
- 구체적인 키워드일수록 정확한 결과 ("AI" < "생성형 AI 규제 동향")
- 정기적으로 같은 주제를 추적하면 트렌드 파악 가능

#### `/51-daily-note` - 일일 기록의 시작점

**언제 사용하나요?**
- **매일 아침**, 하루를 시작할 때 (가장 중요!)
- 오늘 할 일을 정리하고 싶을 때
- 하루 끝에 회고를 작성할 때

**사용 방법:**
```
/51-daily-note
```
> 날짜는 자동으로 오늘 날짜가 입력됩니다.

**생성되는 파일:**
- 위치: `50-periodic/51-daily/2026-01-08.md`
- 파일명 형식: `YYYY-MM-DD.md`

**템플릿 구조 예시:**
```markdown
# 2026-01-08 (수)

## 오늘 할 일
- [ ] 태스크 1
- [ ] 태스크 2

## 진행 중인 프로젝트
(roadmap-builder로 생성된 프로젝트가 있으면 자동 연동)

## 메모
(하루 중 떠오르는 생각들)

## 오늘 한 일
(저녁에 정리)

## 회고
- 잘한 점:
- 개선할 점:
- 내일 할 것:
```

**왜 Daily Note가 중요한가요?**
- 모든 기록의 **진입점** 역할
- 프로젝트 진행상황 자동 추적
- 주간 노트 작성 시 자동으로 참조됨

**팁:**
- 아침에 2분만 투자해서 실행하는 습관 들이기
- 완벽하게 채우려 하지 말고, 빈 칸으로 둬도 OK
- 저녁에 "오늘 한 일"만 간단히 정리

#### `/52-weekly-note` - 주간 회고

**언제 사용하나요?**
- 매주 금요일 퇴근 전 또는 일요일 저녁
- 한 주를 마무리하며 성찰할 때
- 다음 주 계획을 세울 때

**사용 방법:**
```
/52-weekly-note
```
> 해당 주차가 자동으로 계산됩니다.

**생성되는 파일:**

1. 해당 주의 Daily Note들을 스캔

3. 주간 요약 초안 작성

```markdown
# 2026년 2주차 (01.06 - 01.12)

-

## 배운 점
-

## 다음 주 목표
- [ ]

## 감사한 것
-
```

**팁:**
- Daily Note를 꾸준히 쓰면 주간 노트가 자동으로 풍성해짐

## 에이전트 (Agents)

### 에이전트란?

에이전트는 **독립적으로 생각하고 행동하는 전문가 AI**입니다.
슬래시 명령어가 정해진 템플릿을 실행한다면, 에이전트는 대화를 통해 맞춤형 작업을 수행합니다.

**슬래시 명령어 vs 에이전트:**

| 구분 | 슬래시 명령어 | 에이전트 |
|------|--------------|---------|
| 작동 방식 | 정해진 템플릿 실행 | 대화하며 맞춤 작업 |
| 유연성 | 고정된 포맷 | 상황에 맞게 조정 |
| 컨텍스트 | 현재 대화 공유 | 독립적인 작업 공간 |
| 비유 | 자동응답기 | 전문 컨설턴트 |

**언제 에이전트를 쓰나요?**
- 단순 반복이 아닌, **판단이 필요한 작업**
- 여러 파일을 **분석하고 연결**해야 할 때
- **대화형으로** 요구사항을 구체화할 때

### 에이전트 목록

| 에이전트 | 설명 | 호출 방법 |
|----------|------|-----------|
| `roadmap-builder` | 프로젝트 로드맵을 대화형으로 설계 | "로드맵 짜줘", "프로젝트 계획 세워줘" |
| `zettelkasten-linker` | 새 노트에 관련 노트 3개를 자동 제안 | "관련 노트 연결해줘", "연결할 노트 찾아줘" |

### 에이전트 상세 설명

---

#### `roadmap-builder` - 프로젝트 로드맵 설계

**무엇을 하나요?**
새로운 프로젝트를 시작할 때, 대화를 통해 체계적인 로드맵을 만들어줍니다.
막연한 목표를 구체적인 단계와 일정으로 변환합니다.

**언제 사용하나요?**
- 새 프로젝트를 시작하려는데 어디서부터 할지 모를 때
- 데드라인은 있는데 계획이 없을 때
- 큰 목표를 작은 태스크로 쪼개고 싶을 때

**호출 방법:**
```
"퍼스널 브랜딩 웹사이트 프로젝트 로드맵 짜줘. 1월 15일까지 끝내야 해"
"새 프로젝트 계획 세워줘"
"로드맵 짜줘"
```

**대화 예시:**
```
사용자: 블로그 리뉴얼 프로젝트 로드맵 짜줘

에이전트: 블로그 리뉴얼 프로젝트를 시작하시는군요!
          몇 가지 질문드릴게요.

          1. 완료 목표 날짜가 있나요?
          2. 리뉴얼의 주요 목적은 무엇인가요?
          3. 혼자 진행하시나요, 팀이 있나요?

사용자: 1월 말까지, 디자인 변경 위주로, 혼자 해

에이전트: 알겠습니다. 다음과 같은 로드맵을 제안드립니다:

          [1주차] 현황 분석 & 벤치마킹
          [2주차] 디자인 컨셉 확정
          ...
```

**생성되는 결과물:**
- 위치: `10-working/1X-프로젝트명/`
- 구조:
  ```
  10-working/
  └── 13-블로그-리뉴얼/
      ├── 131_프로젝트-개요.md
      ├── 132_로드맵.md
      └── 133_태스크-목록.md
  ```

**주요 기능:**
- **데드라인 역산**: 완료일로부터 거꾸로 일정 계산
- **번호 자동 할당**: Johnny Decimal 체계에 맞춰 폴더/파일 번호 부여
- **Daily Note 연동**: `/51-daily-note` 실행 시 오늘 할 태스크 자동 표시

**팁:**
- 구체적인 데드라인을 알려주면 더 현실적인 계획 수립
- 중간에 계획 변경도 가능 ("2주차 계획 수정해줘")

#### `zettelkasten-linker` - 관련 노트 자동 연결

**무엇을 하나요?**
새로 만든 노트와 연결할 수 있는 기존 노트를 찾아 제안합니다.
단순 키워드 매칭이 아닌, **의미적 관계**를 분석합니다.

**왜 필요한가요?**
- 노트가 쌓이면 어떤 노트끼리 연결되는지 기억하기 어려움
- 연결되지 않은 노트는 사장되기 쉬움
- 예상치 못한 연결에서 새로운 아이디어 발견

**언제 사용하나요?**
- `/21-idea-note`로 아이디어를 기록한 직후
- "이 주제에 대해 예전에 뭔가 썼던 것 같은데..." 할 때
- 지식 그래프를 확장하고 싶을 때

**호출 방법:**
```
"방금 만든 아이디어 노트에 관련 노트 연결해줘"
"이 노트랑 연결할 만한 노트 찾아줘"
"관련 노트 추천해줘"
"볼트 분석해줘"
```

**작동 예시:**
```
사용자: 방금 만든 "AI 회의록 자동화" 노트에 연결할 노트 찾아줘

에이전트: "AI 회의록 자동화" 노트를 분석했습니다.
          관련성 높은 노트 3개를 찾았어요:

          1. "음성인식 API 비교" (21-ideas)
             연결 이유: 회의록 자동화의 핵심 기술인 STT 구현 방안

          2. "생산성 도구 트렌드 2025" (33-news)
             연결 이유: 업무 자동화 시장 동향과 경쟁 제품 분석

          3. "팀 협업 비효율 문제" (21-ideas)
             연결 이유: 이 아이디어가 해결하려는 근본 문제

          노트에 연결을 추가할까요?
```

**연결 유형 예시:**
- **구현/사례**: 이론 ↔ 실제 적용
- **문제-해결**: 문제 정의 ↔ 해결 방안
- **원인-결과**: 배경 ↔ 결론
- **비교/대조**: 유사한 개념들

**핵심 원칙:**
> **모든 연결에는 반드시 "연결 이유"가 있어야 합니다.**

단순히 같은 키워드가 있다고 연결하지 않습니다.
"이 두 노트가 왜 연결되어야 하는가?"에 대한 명확한 답이 있을 때만 제안합니다.

**검색 범위:**
- `20-created/` - 내가 만든 노트
- `30-collected/` - 외부에서 가져온 자료
- **제외**: `50-periodic/` (일일/주간 노트)

**팁:**
- 노트를 만들 때마다 바로 연결 찾기 습관 들이기
- 연결이 3개 미만이면 "더 찾아줘"로 추가 탐색 가능

## 스킬 (Skills)

### 스킬이란?

스킬은 **Claude Code가 자동으로 사용하는 특수 능력**입니다.
외부 서비스나 파일을 다룰 때 상황에 맞게 자동 발동됩니다.

**명령어/에이전트와의 차이점:**

| 구분 | 명령어 | 에이전트 | 스킬 |
|------|--------|---------|------|
| 호출 방법 | `/명령어` 입력 | 키워드 언급 | **자동 감지** |
| 작동 타이밍 | 명시적 요청 시 | 요청 시 | 필요할 때 자동 |
| 비유 | 버튼 누르기 | 전문가 부르기 | 숨겨진 능력 발동 |

**스킬의 특징:**
- 별도로 호출할 필요 없음 (자연어로 요청하면 자동 적용)
- 외부 API, Python 스크립트 등 기술적 작업 처리
- 백그라운드에서 조용히 작동

### 스킬 목록

| 스킬 | 설명 | 자동 발동 상황 | 저장 위치 |
|------|------|---------------|-----------|
| `pdf-reader` | PDF → 텍스트 변환 | PDF 파일 언급 시 | 원본 파일 위치 |
| `youtube-content` | YouTube 자막 추출 | YouTube URL 포함 시 | `30-collected/32-youtube/` |
| `web-scraper` | 웹 페이지 스크래핑 | 웹 크롤링 요청 시 | `30-collected/31-web-scraps/` |
| `notion-down-lv2` | Notion 페이지 동기화 | Notion 연동 요청 시 | `30-collected/36-notion/` |

#### `pdf-reader` - PDF 텍스트 추출

**무엇을 하나요?**
PDF 파일의 텍스트, 표, 이미지를 추출하여 마크다운으로 변환합니다.

**자동 발동 상황:**
```
"이 PDF 요약해줘"
```

**결과물:**

**팁:**
- 스캔된 PDF도 OCR로 텍스트 추출 가능
- `/22-reading-note`와 함께 사용하면 독서 노트 자동 생성

#### `youtube-content` - YouTube 자막 추출

2. 적절한 방식 선택 (BeautifulSoup / Playwright)

**저장 위치:** `30-collected/31-web-scraps/`

#### `pkm-search` - 시맨틱 검색

**무엇을 하나요?**
내 노트들 중에서 의미적으로 관련된 문서를 찾아줍니다.
단순 키워드 매칭이 아닌, 문맥을 이해하는 검색입니다.

**자동 발동 상황:**
```
"내 노트에서 AI 관련 자료 찾아줘"
"예전에 쓴 마케팅 전략 문서 어딨지?"
"기억나? 전에 회의록에서..."
```

**일반 검색 vs 시맨틱 검색:**

| 검색어 | 일반 검색 | 시맨틱 검색 |
|--------|----------|-------------|
| "AI 활용" | 정확히 "AI 활용" 포함 문서 | "인공지능 적용", "머신러닝 도입" 등도 포함 |
| "생산성 높이기" | 해당 문구 포함 문서 | "업무 효율화", "자동화" 등 연관 주제도 검색 |

**팁:**
- 검색어는 질문 형태로도 OK ("어떻게 하면 회의 시간을 줄일 수 있을까?")
- 특정 폴더 한정 검색: "20-created에서만 찾아줘"

#### `notion-down-lv2` / `gdrive-down-lv2` - 외부 서비스 연동

**무엇을 하나요?**
Notion 페이지나 Google Drive 파일을 로컬 폴더로 동기화합니다.

**자동 발동 상황:**
```
"노션 페이지 가져와줘"
"구글 드라이브 폴더 동기화해줘"
"노션에 있는 회의록 다운받아줘"
```

**주요 기능:**
- Notion: 페이지 + 하위 페이지를 마크다운으로 변환
- Google Drive: 폴더 구조 유지하며 다운로드

**저장 위치:** `30-collected/35-gdrive/`, `30-collected/36-notion/` (기본)

**초기 설정 필요:**
- API 키 또는 OAuth 인증 설정
- 상세 가이드: `99-GPTers 20기 학습자료실/` 참고

### 스킬 폴더 구조

각 스킬은 `.claude/skills/[스킬명]/` 폴더에 저장되어 있습니다:

```
.claude/skills/pdf-reader/
├── SKILL.md           ← 스킬 설명, 발동 조건, 사용법
├── requirements.txt   ← Python 패키지 의존성
└── scripts/
    └── pdf_to_text.py ← 실행 스크립트
```

**파일별 역할:**
- `SKILL.md`: Claude가 읽는 스킬 매뉴얼 (언제, 어떻게 사용할지)
- `requirements.txt`: 필요한 Python 라이브러리 목록
- `scripts/`: 실제 실행되는 코드

## 일일 워크플로우

> 이 워크플로우는 **제안**입니다. 자신의 업무 스타일에 맞게 조정하세요.

### 아침 루틴 (5분)

**1. Daily Note 생성**
```
/51-daily-note
```

**2. 오늘 할 일 확인**
- 생성된 Daily Note에서 프로젝트 태스크 확인
- 우선순위 조정 (중요한 것 3개만 선택)

**왜 아침에 Daily Note인가요?**
- 하루의 **의도**를 설정하는 시간
- 어제 미완료 태스크 자동 이월
- 진행 중인 프로젝트 현황 한눈에 파악

---

### 작업 중 (수시)

**아이디어가 떠오를 때:**
```
/21-idea-note [아이디어 제목]
```
→ 2분 안에 캡처하고 본 업무로 복귀

**자료 수집이 필요할 때:**
```
/33-news-briefing [검색 주제]
"이 PDF 요약해줘"
"유튜브 https://... 정리해줘"
```
→ 자동으로 `30-collected/`에 저장

**기존 자료가 필요할 때:**
```
"전에 정리해둔 OO 자료 찾아줘"
```
→ `pkm-search` 스킬이 시맨틱 검색

**노트 연결하기:**
```
"방금 만든 노트에 관련 노트 연결해줘"
```
→ `zettelkasten-linker` 에이전트가 추천

---

### 저녁 루틴 (3분)

**1. Daily Note 업데이트**
- "오늘 한 일" 섹션 간단히 정리
- 완료한 태스크 체크

**팁:**
- 완벽하게 쓰려 하지 말기 (빈 칸 OK)
- 키워드만 적어도 충분
- 3분 넘기지 않기

---

### 주말 루틴 (10분)

**Weekly Note 생성:**
```
/52-weekly-note
```

**자동으로 하는 일:**
- 이번 주 Daily Note 스캔
- "오늘 한 일" 항목 수집
- 주간 요약 초안 작성

**직접 채울 것:**
- 이번 주 가장 의미 있었던 성과
- 배운 점 / 개선할 점
- 다음 주 핵심 목표 (1-3개)

---

### 새 프로젝트 시작 시

**1. 로드맵 설계:**
```
"새 프로젝트 로드맵 짜줘"
```
→ `roadmap-builder` 에이전트와 대화

**2. 프로젝트 폴더 자동 생성:**
- `10-working/1X-프로젝트명/` 자동 생성
- Johnny Decimal 번호 자동 할당

**3. Daily Note 연동:**
- 이후 `/51-daily-note` 실행 시 프로젝트 태스크 자동 표시

## 커스터마이징

이 워크스페이스는 **템플릿**입니다. 자신의 업무에 맞게 수정하세요.

### 커스터마이징 원칙

1. **먼저 기본 구조로 2주 사용해보기**
   - 바로 수정하지 말고, 어떤 게 불편한지 파악

2. **불편한 점이 생기면 그때 수정**
   - "이 폴더는 안 쓰게 되네" → 삭제 또는 통합
   - "이런 종류의 노트가 자주 생기네" → 새 하위 폴더 추가

3. **CLAUDE.md도 함께 업데이트**
   - 폴더 구조를 바꾸면 반드시 `.claude/CLAUDE.md` 수정
   - AI가 새 구조를 이해해야 제대로 작동

### 새 슬래시 명령어 추가하기

**예시:** `/23-meeting-note` 회의록 명령어 추가

**1단계: 폴더 생성**
```
20-created/23-meetings/
```

**2단계: 명령어 파일 생성**
`.claude/commands/23-meeting-note.md` 파일 생성

**3단계: 템플릿 생성**
`00-system/01-templates/23-meeting-template.md` 파일 생성

**4단계: CLAUDE.md 업데이트**
명령어 목록 테이블에 새 명령어 추가
